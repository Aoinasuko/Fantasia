Fantasia distributed DLL / runtime license list
Last updated: 2026-06-17

Scope
-----
This file summarizes third-party runtime binaries that may be bundled under
publish/.fantasia_runtime by scripts/build_exe.ps1.

It covers DLL files, Python extension modules (.pyd), and helper executables
used as runtime components. Model files under model/text and model/graphic are
not bundled by default and are not covered by this file.

This is a practical distribution checklist, not legal advice. Before a public
release, verify the current upstream license terms for the exact binary package
you ship.

Runtime profile notes
---------------------

The exact bundled runtime folders depend on the build profile:

- cpu: bin/llama and bin/stable-diffusion.cpp
- vulkan: bin/llama-vulkan or bin/llama fallback, and bin/stable-diffusion.cpp-vulkan or bin/stable-diffusion.cpp fallback
- cuda: bin/llama-cuda and bin/stable-diffusion.cpp-cuda
- all: all available runtime folders

CUDA builds duplicate NVIDIA runtime DLLs in both llama-cuda and
stable-diffusion.cpp-cuda when each upstream binary package ships its own copy.


Local full-text notices currently bundled
-----------------------------------------

The build keeps upstream notice files that are already present in runtime
folders. Current examples:

- .fantasia_runtime/bin/stable-diffusion.cpp*/stable-diffusion.cpp.txt
- .fantasia_runtime/bin/stable-diffusion.cpp*/ggml.txt
- .fantasia_runtime/tk/license.terms

If a runtime binary folder is replaced, refresh this file and keep any upstream
LICENSE, COPYING, NOTICE, or EULA files that come with the replacement package.
